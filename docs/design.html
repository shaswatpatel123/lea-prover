Lea — Architecture

A config-driven Lean-proving agent with one config-in / event-out contract. The agent is the product; the CLI, a UI, and eval are consumers of the same core.

See decisions.md for the rationale and the mini-swe-agent mapping.

Architecture flow

Consumers

cli.py
--config + flag overrides
Teammate UI
renders events live
eval/ harnesses
batch, collect transcript
→

Config in

default.yaml
single source of defaults
config.load_config
base + --config overlay
validation.validate_config
I/O-free, typed errors → LeaConfig
→

Agent core

agent.run_events
generator loop — yields events
providers.stream
LiteLLM, stream or blocking
tools.TOOL_HANDLERS
read/write/edit/lean_check/bash/search
→

Events out

typed event stream
TextDelta · ToolCall · Usage · Finished
render_to_stdout
CLI transcript + per-turn cost
UI / eval consumers
same events, own rendering

Key property: the LiteLLM engine swap and the streaming/blocking choice never touch the loop or the consumers — providers.stream() always yields the same event types, so everything to the right of it is invariant. The UI can also call validate_config() directly on a payload (dashed path — no disk, no agent run).

Turn lifecycle

run_events → consumer
TurnStarted(turn)
run_events → providers.stream → LiteLLM
stream(model, msgs, tools, model_kwargs, streaming)
providers → run_events
TextDelta*ToolCall (+_ToolMeta)Done(usage, cost)
run_events → consumer
AssistantTextDelta*ToolCalled*UsageUpdated(in, out, cost) ← per-turn cost
if tool calls present
run each TOOL_HANDLERS[name](args) → ToolResulted, persist session, next turn
else (no tool calls, or max_turns hit)
Finished(reason, text, turns, usage, cost, transcript)  reason ∈ {completed, max_turns}

Config schema

model:
  name: gemini/gemini-3.1-pro-preview   # LiteLLM provider/model
  stream: true                          # true → live tokens; false → one blocking call
  model_kwargs:                         # open passthrough to litellm.completion
    max_tokens: 16384
agent:
  prompt_variant: default
  max_turns: null                       # null → run until done
